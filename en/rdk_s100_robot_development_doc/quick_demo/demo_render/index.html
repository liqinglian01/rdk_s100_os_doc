2.2 Data visualization

Web display​

Overview​

Web display previews camera images (JPEG) and algorithm results, streams them over the network to a PC browser for rendering. The viewer can also show video only, without rendering intelligence overlays.

Code repository: https://github.com/D-Robotics/hobot_websocket

Supported platforms​

PlatformRuntimeExample
RDK X3, RDK X3 ModuleUbuntu 20.04 (Foxy), Ubuntu 22.04 (Humble)Start MIPI camera and show image in Web
RDK X5, RDK X5 Module, RDK S100Ubuntu 22.04 (Humble)Start MIPI camera and show image in Web
RDK UltraUbuntu 20.04 (Foxy)Start MIPI camera and show image in Web
x86Ubuntu 20.04 (Foxy)Start USB camera and show image in Web

Prerequisites​

RDK platform​

  1. Confirm the F37 camera is connected to the RDK correctly.

  2. Confirm the PC can reach the RDK over the network.

  3. Confirm TogetheROS.Bot is installed.

x86 platform​

  1. Confirm the system is Ubuntu 20.04 and tros.b is installed.

  2. Confirm the USB camera is plugged in and recognized.

Usage​

RDK platform​

  1. SSH into the RDK and start the board-side programs:

    a. Start mipi_cam

    • Foxy
    • Humble
    # Configure tros.b
    source /opt/tros/setup.bash
    # Configure tros.b
    source /opt/tros/humble/setup.bash
    ros2 launch mipi_cam mipi_cam.launch.py mipi_video_device:=F37

    b. Start encoding

    • Foxy
    • Humble
    # Configure tros.b
    source /opt/tros/setup.bash
    # Configure tros.b
    source /opt/tros/humble/setup.bash
    ros2 launch hobot_codec hobot_codec_encode.launch.py

    c. Start WebSocket

    • Foxy
    • Humble
    # Configure tros.b
    source /opt/tros/setup.bash
    # Configure tros.b
    source /opt/tros/humble/setup.bash
    ros2 launch websocket websocket.launch.py websocket_image_topic:=/image_jpeg websocket_only_show_image:=true

  2. In a PC browser (Chrome/Firefox/Edge), open http://IP:8000 to view the image, where IP is the RDK address.

x86 platform​

  1. Start the hobot_usb_cam node

    • Foxy
    • Humble
    # Configure tros.b
    source /opt/tros/setup.bash
    # Configure tros.b
    source /opt/tros/humble/setup.bash
    # Set usb_video_device to your actual USB camera device node
    ros2 launch hobot_usb_cam hobot_usb_cam.launch.py usb_image_width:=1280 usb_image_height:=720 usb_video_device:=/dev/video0

  2. Start the WebSocket node

    • Foxy
    • Humble
    # Configure tros.b
    source /opt/tros/setup.bash
    # Configure tros.b
    source /opt/tros/humble/setup.bash
    ros2 launch websocket websocket.launch.py websocket_image_topic:=/image websocket_only_show_image:=true

  3. In a PC browser (Chrome/Firefox/Edge), open http://IP:8000 to view the stream. Use the PC’s IP, or localhost when browsing on the same machine.

Notes​

  1. WebSocket uses port 8000. If it is in use, startup fails. Mitigations:

    • Run lsof -i:8000 to find the process, then kill <PID> and restart WebSocket.

    • Or edit /opt/tros/${TROS_DISTRO}/lib/websocket/webservice/conf/nginx.conf, change listen to an unused port above 1024, and use the matching URL in the browser.

RViz2 display​

Overview​

TogetheROS.Bot supports ROS 2 Foxy/Humble. To preview images, subscribe with RViz2.

Supported platforms​

PlatformRuntime
RDK X3, RDK X3 ModuleUbuntu 20.04 (Foxy), Ubuntu 22.04 (Humble)
RDK X5, RDK X5 Module, RDK S100Ubuntu 22.04 (Humble)
RDK UltraUbuntu 20.04 (Foxy)

Prerequisites​

RDK platform​

  1. RDK is flashed with Ubuntu 20.04 or 22.04.

  2. tros.b is installed on the RDK.

  3. The PC runs Ubuntu 20.04/22.04 with ROS 2 Foxy/Humble desktop and RViz2, on the same subnet as the RDK (first three octets of the IP match).

    • ROS 2 install: Foxy, Humble

    • RViz2 on PC: sudo apt install ros-$ROS_DISTRO-rviz-common ros-$ROS_DISTRO-rviz-default-plugins ros-$ROS_DISTRO-rviz2 where $ROS_DISTRO is foxy or humble.

Usage​

RDK platform​

  1. SSH into the board and start the programs

    • Foxy
    • Humble
    # Configure tros.b
    source /opt/tros/setup.bash
    # Configure tros.b
    source /opt/tros/humble/setup.bash
    # Start F37 camera publishing BGR8
    ros2 launch mipi_cam mipi_cam.launch.py mipi_out_format:=bgr8 mipi_image_width:=480 mipi_image_height:=272 mipi_io_method:=ros mipi_video_device:=F37

    Note: Do not change mipi_out_format arbitrarily. RViz2 supports RGB8, RGBA8, BGR8, BGRA8, etc.

  2. If you see output like below, the node started successfully:

    [INFO] [launch]: All log files can be found below /root/.ros/log/2022-08-19-03-53-54-778203-ubuntu-2881662
    [INFO] [launch]: Default logging verbosity is set to INFO
    [INFO] [mipi_cam-1]: process started with pid [2881781]

  3. On the RDK, open another terminal and list topics:

    • Foxy
    • Humble
    # Configure tros.b
    source /opt/tros/setup.bash
    # Configure tros.b
    source /opt/tros/humble/setup.bash
    # List topics
    ros2 topic list

    Output:

    /camera_info
    /image_raw
    /parameter_events
    /rosout

  4. On the PC, list topics:

  • Foxy
  • Humble
source /opt/ros/foxy/setup.bash
source /opt/ros/humble/setup.bash
# Configure ROS 2
ros2 topic list

Output:

/camera_info
/image_raw
/parameter_events
/rosout
  1. On the PC, subscribe and preview the camera:
  • Foxy
  • Humble
source /opt/ros/foxy/setup.bash
source /opt/ros/humble/setup.bash
# Configure ROS 2
ros2 run rviz2 rviz2

In RViz2, click Add, pick the published image topic (here /image_raw), then choose Image:

Example result:

Notes​

  1. If ros2 topic list on the PC does not show the camera topics:

    • Check the RDK is publishing:

      • Foxy
      • Humble
      # Configure tros.b
      source /opt/tros/setup.bash
      # Configure tros.b
      source /opt/tros/humble/setup.bash
      ros2 topic list

      Output:

      /camera_info
      /image_raw
      /parameter_events
      /rosout

    • Ping between PC and RDK.

    • Confirm PC and RDK share the same /24 (first three octets).

Foxglove display​

Overview​

Foxglove is an open-source toolkit (web and desktop) for robotics development and debugging.

This section uses recording and playback: record ROS 2 topics to a file for later analysis—useful for troubleshooting, tuning, and debugging.

The demo uses the hobot_visualization package to turn inference results into ROS 2 visualization topics.

Code repository: https://github.com/D-Robotics/hobot_visualization

Supported platforms​

PlatformRuntime
RDK X3, RDK X3 ModuleUbuntu 20.04 (Foxy), Ubuntu 22.04 (Humble)
RDK X5, RDK X5 Module, RDK S100Ubuntu 22.04 (Humble)
x86Ubuntu 20.04 (Foxy)

Prerequisites​

RDK platform​

  1. F37 camera is connected correctly.

  2. PC can reach the RDK over the network.

  3. TogetheROS.Bot is installed.

x86 platform​

  1. Ubuntu 20.04 with tros.b installed.

Usage​

RDK platform / x86 platform​

  1. SSH into the RDK and start the board programs:
  • Foxy
  • Humble
# Configure tros.b
source /opt/tros/setup.bash
# Configure tros.b
source /opt/tros/humble/setup.bash
export CAM_TYPE=fb
cp -r /opt/tros/${TROS_DISTRO}/lib/dnn_node_example/config/ .

ros2 launch hobot_visualization hobot_vis_render.launch.py

In another SSH session, record topics:

  • Foxy
  • Humble
# Configure tros.b
source /opt/tros/setup.bash
# Configure tros.b
source /opt/tros/humble/setup.bash
# Record rosbag in the current working directory
ros2 bag record -a
  1. Play the bag in Foxglove Studio online

  1. Open https://foxglove.dev/studio in Chrome/Firefox/Edge.

Note: First-time use requires sign-in (Google or email).

  1. Enter the visualization UI

  2. Select a local rosbag file

  3. Open the layout, click settings (top right), enable marker rendering for playback

  4. Play

  5. View the data

Notes​

  1. For image panels, use official ROS 2 message types and encodings supported by Foxglove. See Image panel docs.

  2. Recording may pick up other hosts’ traffic. To isolate, set e.g. export ROS_DOMAIN_ID=1.

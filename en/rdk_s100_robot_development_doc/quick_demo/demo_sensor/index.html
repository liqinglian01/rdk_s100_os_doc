2.1 Data acquisition

USB image capture​

Overview​

Cameras are commonly used for environmental perception. USB cameras are easy to obtain and use. TogetheROS.Bot supports USB cameras and standard ROS 2 image messages.

Code repository: https://github.com/D-Robotics/hobot_usb_cam.git

Supported platforms​

PlatformRuntime
RDK X3, RDK X3 ModuleUbuntu 20.04 (Foxy), Ubuntu 22.04 (Humble)
RDK X5, RDK X5 ModuleUbuntu 22.04 (Humble)
RDK S100, RDK S100PUbuntu 22.04 (Humble)
RDK X5, RDK X5 ModuleUbuntu 22.04 (Humble)
RDK S100, RDK S100PUbuntu 22.04 (Humble)
RDK UltraUbuntu 20.04 (Foxy)
x86Ubuntu 20.04 (Foxy)

Prerequisites​

RDK platform​

  1. Confirm the USB camera works and plug it into the RDK.

  2. RDK is flashed with Ubuntu 20.04 or 22.04.

  3. tros.b is installed on the RDK.

  4. The PC can reach the RDK over the network.

x86 platform​

  1. Confirm the USB camera works and plug it into the PC or server.

  2. Ubuntu 20.04 is installed with tros.b.

Usage (default usb_pixel_format is mjpeg)​

    source /opt/tros/setup.bash
    # Configure tros.b

Notes​

  1. Calibrate the camera and set the calibration file path to publish camera_info; without it, intrinsics are not published but other features still work.

  2. To set the calibration path:

    • Foxy
    • Humble
    # Configure tros.b
    source /opt/tros/setup.bash
    # Configure tros.b
    source /opt/tros/humble/setup.bash
    # Launch
    ros2 launch hobot_usb_cam hobot_usb_cam.launch.py usb_camera_calibration_file_path:=/absolute/path/to/your/calibration.yaml

  3. On x86, if Ubuntu 20.04 runs in a VM, set USB compatibility to USB 3.1 in the VM USB controller settings.

MIPI image capture​

Overview​

For perception, robots often use cameras and ToF sensors. TogetheROS.Bot wraps common sensors in the hobot_sensor module with standard ROS image messages. If configured parameters do not match the attached camera, the driver adapts. Supported MIPI sensors:

#NameImageSpecPlatformsLink
1F372 MPRDK X3, RDK X3 ModuleF37
2GC46634 MPRDK X3, RDK X3 ModuleGC4663
3IMX2198 MPRDK X3, RDK X3 Module, RDK Ultra, RDK X5, RDK X5 Module, RDK S100IMX219
4IMX4772 MPRDK X3, RDK X3 ModuleIMX477
5OV56472 MPRDK X3, RDK X3 Module, RDK X5, RDK X5 ModuleOV5647
6IMX4152 MPRDK X5, RDK X5 ModuleIMX415

Code repository: https://github.com/D-Robotics/hobot_mipi_cam.git

Prerequisites​

RDK platform​

  1. Connect the camera correctly, e.g. IMX219 on RDK S100:

  2. RDK is flashed with Ubuntu 20.04 or 22.04.

  3. tros.b is installed.

  4. The PC can reach the RDK over the network.

Usage​

RDK platform​

How to acquire and preview data:

  1. SSH into the RDK.

  2. Start hobot_sensor:

    • Foxy
    • Humble
    # Configure tros.b
    source /opt/tros/setup.bash
    # Configure tros.b
    source /opt/tros/humble/setup.bash
    # Launch
    ros2 launch mipi_cam mipi_cam.launch.py

  3. If you see output like below, the node started successfully:

    [INFO] [launch]: All log files can be found below /root/.ros/log/2022-06-11-15-16-13-641715-ubuntu-8852
    [INFO] [launch]: Default logging verbosity is set to INFO
    [INFO] [mipi_cam-1]: process started with pid [8854]
    ...

  4. For Web preview of F37 raw data, encode to JPEG and start WebSocket in two terminals:

    • Foxy
    • Humble
    # Configure tros.b
    source /opt/tros/setup.bash
    # Configure tros.b
    source /opt/tros/humble/setup.bash
    # Encoder
    ros2 launch hobot_codec hobot_codec_encode.launch.py

    # Another terminal with tros.b sourced — WebSocket
    ros2 launch websocket websocket.launch.py websocket_image_topic:=/image_jpeg websocket_only_show_image:=true

  5. Open http://IP:8000 in a browser (RDK IP), use the Web viewer for the live F37 feed.

  6. On the PC, read camera intrinsics (values depend on your calibration file):

  • Foxy
  • Humble

root@ubuntu:~# source /opt/ros/humble/setup.bash

Notes​

  1. mipi_cam ships calibration for F37 and GC4663; default is F37 (F37_calibration.yaml). For GC4663, set the path:

    • Foxy
    • Humble
    # Configure tros.b
    source /opt/tros/setup.bash
    # Configure tros.b
    source /opt/tros/humble/setup.bash
    # Launch
    ros2 launch mipi_cam mipi_cam.launch.py mipi_video_device:=GC4663 mipi_camera_calibration_file_path:=/opt/tros/${TROS_DISTRO}/lib/mipi_cam/config/GC4663_calibration.yaml

  2. Do not hot-plug the camera while power is on—risk of damaging the module.

  3. If hobot_sensor fails to start:

    • Check hardware
    • Confirm tros.b is sourced
    • Verify parameters per README.md

  4. Two streams on the same topic conflict; remap for a second camera (X5/S100 only):

    • Foxy
    • Humble
    # Configure tros.b
    source /opt/tros/setup.bash
    # Configure tros.b
    source /opt/tros/humble/setup.bash
    # Run
    ros2 run mipi_cam mipi_cam --ros-args --remap /image_raw:=/image_raw_alias
    # Or launch
    ros2 launch mipi_cam mipi_cam_topic_remap.launch.py

Stereo MIPI capture​

Overview​

For stereo perception, robots may use stereo cameras or ToF. TogetheROS.Bot wraps sensors in hobot_sensor with standard ROS image messages. Supported stereo MIPI types:

TypeModelSpecPlatforms
CameraSC230ai2 MPRDK X5, RDK X5 Module, RDK S100, RDK S100P
CameraSC132gs2 MPRDK X5, RDK X5 Module, RDK S100, RDK S100P

Code repository: https://github.com/D-Robotics/hobot_mipi_cam.git

Supported platforms​

PlatformRuntime
RDK X5, RDK X5 ModuleUbuntu 22.04 (Humble)
RDK 100, RDK S100PUbuntu 22.04 (Humble)
RDK 100, RDK S100PUbuntu 22.04 (Humble)

Prerequisites​

RDK platform​

  1. Connect the stereo camera, e.g. SC230ai on S100:

  2. RDK is flashed with Ubuntu 22.04.

  3. tros.b is installed.

  4. The PC can reach the RDK over the network.

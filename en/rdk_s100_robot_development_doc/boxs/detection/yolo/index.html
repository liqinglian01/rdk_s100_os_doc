YOLO

Overview​

The YOLO object-detection example takes images as input, runs inference on the BPU, and publishes messages with class labels and bounding boxes. Supported variants include YOLOv2, YOLOv3, Ultralytics YOLOv5, YOLOv5x, Ultralytics YOLOv8, YOLOv10, YOLOv11, YOLOv12, and more.

Models are trained on the COCO dataset with 80 classes (people, animals, fruit, vehicles, etc.).

You can train on custom data with Ultralytics: (https://docs.ultralytics.com/zh/modes/train)

Code repository: (https://github.com/D-Robotics/hobot_dnn)

Use cases: YOLO is a fast, general one-stage detector—suitable for waste sorting, vehicle detection, etc., in ADAS, smart home, and similar domains.

Vehicle detection example: (https://github.com/JunshengFu/vehicle-detection)
Fall detection example: (https://github.com/xiaobin1231/Fall-Detection-By-YOLOV3-and-LiteFlowNet)

Prerequisites​

RDK platform​

  1. RDK is flashed with Ubuntu 20.04 or 22.04.

  2. TogetheROS.Bot is installed.

  3. MIPI or USB camera is available; without a camera you can use JPEG/PNG files or MP4/H.264/H.265 video for offline feed.

  4. The PC can reach the RDK over the network.

x86 platform​

  1. Ubuntu 20.04 is installed.

  2. tros.b is installed.

Usage​

RDK platform​

Publish from MIPI camera​

The YOLOv2 example subscribes to MIPI images, publishes detection messages, and uses the WebSocket package to render in a PC browser.

  • Foxy
  • Humble
# Configure tros.b
source /opt/tros/setup.bash
# Configure tros.b
source /opt/tros/humble/setup.bash
# MIPI camera
export CAM_TYPE=mipi

# Launch
ros2 launch dnn_node_example dnn_node_example.launch.py dnn_example_config_file:=config/yolov2workconfig.json dnn_example_image_width:=1920 dnn_example_image_height:=1080

Publish from USB camera​

The YOLOv2 example subscribes to USB images, publishes detection messages, and uses WebSocket for browser rendering.

  • Foxy
  • Humble
# Configure tros.b
source /opt/tros/setup.bash
# Configure tros.b
source /opt/tros/humble/setup.bash
# USB camera
export CAM_TYPE=usb

# Launch
ros2 launch dnn_node_example dnn_node_example.launch.py dnn_example_config_file:=config/yolov2workconfig.json dnn_example_image_width:=1920 dnn_example_image_height:=1080

Local image feed​

The YOLOv2 example feeds local JPEG/PNG images and saves rendered results under the working directory.

  • Foxy
  • Humble
# Configure tros.b
source /opt/tros/setup.bash
# Configure tros.b
source /opt/tros/humble/setup.bash
# Launch
ros2 launch dnn_node_example dnn_node_example_feedback.launch.py dnn_example_config_file:=config/yolov2workconfig.json dnn_example_image:=config/target.jpg

Besides YOLOv2, other YOLO variants are supported. Switch with dnn_example_config_file, e.g. YOLOv3: dnn_example_config_file:="config/yolov3workconfig.json", YOLOv5: config/yolov5workconfig.json, YOLOv8: config/yolov8workconfig.json, YOLOv10: config/yolov10workconfig.json, YOLOv11: config/yolov11workconfig.json, YOLOv12: config/yolov12workconfig.json.

x86 platform​

Local image feed​

The YOLOv2 example feeds local JPEG/PNG images and saves rendered results locally.

# Configure tros.b
source /opt/tros/setup.bash

# Launch
ros2 launch dnn_node_example dnn_node_example_feedback.launch.py dnn_example_config_file:=config/yolov2workconfig.json dnn_example_image:=config/target.jpg

Besides YOLOv2, YOLOv3 is supported; YOLOv5 is not on x86. Switch with dnn_example_config_file, e.g. YOLOv3: dnn_example_config_file:="config/yolov3workconfig.json".
